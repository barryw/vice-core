# vice-core — canonical shared VICE 3.10.0 core (WHI)

This repository is the **single upstream-tracking point** for the VICE emulator
core shared by Walker Heavy Industries' two VICE forks:

- **[barryw/vice-mcp](https://github.com/barryw/vice-mcp)** — the MCP engine build
  (`--enable-gtk3ui --enable-mcp-server`).
- **[barryw/vice-macos](https://github.com/barryw/vice-macos)** — the macOS app build
  (`--enable-macosui`).

The tree at the repo **root** is the VICE source tree (VICE **3.10.0**), carrying
**both** product subtrees behind configure flags:

- `src/mcp/` — MCP server module, gated by `--enable-mcp-server` (**default = no**).
- `src/arch/macos/` — native macOS UI/runtime, gated by `--enable-macosui`.

Rationale, mechanism and the go/no-go decision: **ADR 0004** (VICE core convergence)
and **WAL-78 / WAL-79** in the WHI issue tracker. The empirical 3-way-merge proof
that the two forks are orthogonal lives in [`MERGE-ANALYSIS.md`](MERGE-ANALYSIS.md).

## Why one core

Both forks pin upstream VICE 3.10.0 and diverge in **orthogonal** subtrees, so an
upstream SVN bump previously had to be vendored **twice**. This repo makes it a
**single** sync. Downstream product repos consume `vice/` from here via
`git subtree pull --squash` (mirrors the existing SVN-vendor pattern; no submodule
friction with config-service/Woodpecker).

## Dual-flag CI gate (both-green-before-pull)

Every upstream bump MUST build **both** flag combos green before any downstream
subtree-pull may land — this prevents silent bit-rot of an unbuilt flag combo
(the real risk; the cores themselves don't conflict):

- [`.woodpecker/build-engine.yaml`](.woodpecker/build-engine.yaml) — Linux,
  `--enable-gtk3ui --enable-mcp-server` (+ a headless combo).
- [`.woodpecker/build-macos.yaml`](.woodpecker/build-macos.yaml) — darwin/arm64,
  `--enable-macosui`.

## Phase-A boundary (hard)

Through Phase A, VICE Mac keeps `--enable-mcp-server` **default = no**, so **zero**
new dependency (no libmicrohttpd, no `network.server` entitlement) enters the
signed/notarized Mac bundle. libmicrohttpd / entitlement / security work is
deferred to Phase B (F4b).

## Source-only convention

Autotools artifacts (`configure`, `*/Makefile.in`, `aclocal.m4`) and the generated
bison parser (`src/monitor/mon_parse.c`) are **regenerated by CI** (`./autogen.sh`,
`bison`), not committed. `src/monitor/mon_parse.y` is the grammar source of truth.

## Reconstruction

The merged tree is reproducibly regenerated by
`tools/vice-core/construct-core.sh` in the **whi-brand** repo. See that script and
its README for the exact recipe.
